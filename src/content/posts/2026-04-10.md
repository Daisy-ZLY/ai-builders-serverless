**原文引用：**

>
> "Everyone is substantially underestimating the total demand for software and automation in areas that don't feel like 'software'... Most companies have not been able to bring automation to most areas of work because it's been too complex or costly... Now that agents bring down the cost, companies will light these projects up."

**大白话：** 大多数人在用 AI 聊天——有用，但天花板低。真正的暴击是让 Agent 在后台跑长任务，产出率可以 200%+，而且没有上限。更大的故事是：绝大多数公司从来没有能力自动化他们的工作流——不是不想，是太贵太难。AI Agent 把成本打下来了，一大堆"从没被做过"的项目要批量点亮了。

**中文解读：** Box CEO Levie 在连续三条推文中构建了一个完整的框架：(1) "双城记"——聊天式 AI 和 Agent 式 AI 的产出差距是数量级的，但目前绝大部分用户还停留在聊天端；(2) "隐藏需求"——非技术公司（零售、制药、医疗）积压了十年的自动化需求，以前因为成本和技术限制做不了，现在 Agent 同时解决了这两个瓶颈；(3) "落地挑战"——合规管理、安全、上下文供给、遗留系统等"棘手问题"会让升级过程很慢，但这恰恰创造了巨大的新职位和新平台机会。

- 🔴 但另一面是：传统企业的数据孤岛、合规要求、组织惰性不会因为 Agent 来了就自动消失，"点亮"和"落地"之间可能有 3-5 年的鸿沟

**和你有关：** 如果你是技术人员，最大的金矿可能不在科技公司，而在帮传统行业（制药、零售、医疗、制造）落地 AI 自动化。另外，如果你还只是在"和 AI 聊天"，是时候思考怎么让 Agent 在后台为你跑长任务了。

**原文引用：**
> "Allowing Sonnet to 'phone a friend' (i.e. call Opus) increases performance while also reducing total cost since it reduces tokens spent trying to solve more complex tasks."

> "In evals, Sonnet with an Opus advisor scored 2.7 percentage points higher on SWE-bench Multilingual than Sonnet alone, while costing 11.9% less per task."
>
> "Add the advisor tool to your Messages API call. When your Sonnet or Haiku agent hits a hard decision mid-run, it consults Opus, gets a plan, and continues, all within a single API request."

**大白话：** Anthropic 搞了个"考试时旁边坐学霸"机制——Sonnet 遇到难题就问 Opus，效果更好，还更便宜。一切都在一次 API 请求里完成。

**中文解读：** Anthropic 推出了 Advisor Tool（beta），在 Messages API 中让 Sonnet/Haiku 在遇到复杂决策时自动咨询 Opus，获得规划建议后继续执行。SWE-bench Multilingual 上 +2.7pp 的提升伴随 -11.9% 的成本——完美体现了"小模型做执行、大模型做决策"的分工哲学。技术实现上，这一切发生在单个 API 请求内部，开发者不需要自己搭建路由逻辑。这可能是目前最优雅的"模型路由"方案。

**辩证视角：**

- 🔴 但另一面是：这依然是 Anthropic 的封闭方案。如果其他模型提供商不跟进，跨供应商的 Agent 系统怎么办？

**和你有关：** 如果你在用 Anthropic API 做 Agent 开发，立刻尝试 Advisor Tool。把 Opus 当"技术总监"，Sonnet 当"执行工程师"——成本和效果可以兼得。

### Unsupervised Learning Ep 84 — OpenAI 首席科学家 Jakub Pachocki 深度访谈

**节目：** Unsupervised Learning（Redpoint AI 播客）
**嘉宾：** Jakub Pachocki（OpenAI Chief Scientist）
**主持：** Jacob Efron（Redpoint Ventures）

**深度摘要：**

OpenAI 首席科学家罕见地公开讨论了 AI 研究的现状和方向。以下是最关键的信息点：

**1. 自动化 AI 研究员时间线**
- 目标：2026年9月实现"研究实习生级别"，2028年3月实现"全自动 AI 研究员"
- 关键区分：能力差异在于"自主工作的时间跨度"和"任务的模糊程度"
- 预计很快会有能自主工作"几天"的模型，使用比现在多得多的计算资源

**2. RL 在更广泛领域的前景**
- 代码/数学成功原因：可验证奖励信号 + 高 B2B 价值 → 资源倾斜
- 更广领域的挑战 ≈ 长时间任务的挑战 → 核心是"如何评估部分进展"
- 建议：企业不一定需要自己跑 RL，搞清楚 eval 和数据、喂入上下文可能就够了

**3. Continual Learning 的冷思考**
- "我有点困惑"——GPT-3 论文的兴奋点就是这类模型能持续学习
- 当前路径（预训练 + RL）就是通向持续学习的路，不需要另起炉灶

**4. AI for Science**
- GPT 5.2 Pro 已经在 AI 研究中产生了一些"小但有影响力"的新想法
- "看到模型在一小时内想出我需要一两周才能想到的想法——非常奇怪的感觉"

**5. Chain of Thought 与 AI 安全**
- Pachocki 最看重的安全方向：CoT 没有被直接训练来"表演"，能揭示模型真实意图
- 长期价值对齐的核心挑战是"泛化"——模型在陌生情况下回退到什么价值观？

**和你有关：**
- 企业不需要自己训练模型，把 eval 和领域数据准备好喂入上下文就行
- 模型很快能自主工作数天，但"给它正确的上下文和工具访问"仍是最大瓶颈
- "能与 AI 协作完成复杂研究"将成为最稀缺的技能之一
