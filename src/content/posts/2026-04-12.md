**原文：**
Thin harness, fat skills: If your memory dies when your harness dies, you built the harness too thick. Memory is markdown. Skills are markdown. Brain is a git repo. The harness is a thin conductor — it reads the files, it does the work.

**大白话：** Agent 架构应遵循「薄线束、厚技能」原则，记忆和技能用 markdown + git 持久化，框架只做薄导体

**中文解读：** Garry Tan 在三个月的开源实践后提出了 Agent 架构的核心理念: harness（线束/框架）应该做得尽可能薄，技能和记忆才是核心资产。记忆是 markdown，技能是 markdown，大脑是 git repo。如果框架挂了记忆也跟着消失，说明你的框架太厚了。这直接挑战了当前主流的「重框架」Agent 设计思路。

🟢 这种设计让 Agent 的知识和记忆脱离了具体框架的绑定，实现了真正的可移植性和持久性，降低了迁移成本
